Dracomaton is a top-down roguelike bullet hell where you swap between multiple playstyles on the fly. Equip yourself with various Forms that have different strengths and weaknesses, or Mod existing Forms with the characteristics of other ones!

What part did I do?

  • Item System/Item
    • In Dracomaton, an item is an object that can have negative or positive effects on player which adds a lot of variances to gameplay.

    I was responsible for initializing the framework of item system, including how to create a new item; how to make each item has its unique effect; how to apply item’s effect to player etc.
    To resolve those problems, I have used Unity’s ScriptableObjects to create a base class: BaseItem, which all the future items can inherit from. Each base item has its own info such as id, icon, and Item Stats Mods & Item Status Mods are the part that takes care of item’s unique effect. The way it works is that when the Apply function in base item script get called, it will add all the mods(aka. effects) to its host damage source/damageable, mostly player.

    Considering that designers will collaborate on items throughout the development, I created a designer-friendly system so they can easily test and tweak the item strength during the test by simply changing the attributes on the inspector.

  • Resources Management System (Rewards, Shops)
    • In Dracomaton, random objects need to be spawned on the map under different situations, for example, shop items are randomly selected from all the items in the game. Therefore, I created a Resource Management System to organize all the rewards/resources in the game to work as a general helper function, including every type of resource, such as UI icons, Damage Text, and even VFX. Both the Shop system and Reward system utilized the Resource Manager.

    Resources Manager

    To explain how the resource management system works, the ResourcesManager.cs is a singleton class that fetches resource objects (prefabs) from the project folders to separate lists when awake.

    For code clarity, I made all the individual managers as a partial class of ResourcesManager.cs, which allows each resource type to have its own manager to make the code more readable and generate a random object more easily by just calling the function in the corresponding manager-script. Since each manager script is individual, programmers can create many single manager scripts without complicated primary resources manager script.

    Reward System

    For the reward system, In Dracomaton, players will get different rewards (Items, Forms etc.) based on the room they entered and finished, for example, clearing a health room will reward the player a health portion.

    Various room types will emerge in the game loop as the game progresses. When a new room is loaded, the reward spawner script will decide the reward type from the current room’s type (stored in the room controller). After the player clears the room, the reward script will detect the tiles on the tilemap in a specific range to ensure the reward spawns in a reachable place (not water). If the reward type is items or forms, the reward will be the items/objects randomly returned by their manager script.

    Various room types will emerge in the game loop as the game progresses. Therefore, I created a resource management system to organize all the rewards/resources in the game to work as a general helper function, including every type of resource, such as UI icons, Damage Text, and even VFX.

    Shop System

    I was responsible for offering three shop items (mostly the same object as rewards) in the shop room for players. One of the three items is always a Health portion for game balance reason and the other two can be randomly chosen from any possible objects.

    The way I fulfill these is by making a shop item position prefab that has been preset a spawning method, which allows it to be easily moved to any position on the screen and it is. For example, if the item type is Random, it means this position will spawn a random item, otherwise, it will only spawn the selected type of item. Once the shop room is awake, the shop room manager will populate three shop item positions in the game.
  • UI

    • Mod Menu

    Mod Menu is the UI that is highly releated to Dracomaton's core gameplay, Forms and Mods. There are two types of slots in the game: the upper slots are Form slots, and the lower slots are Mods slots. The mechanics of Dracomaton is that players can switch the form and mod by swapping their positions. For example, if players drag Fire Form to the Construct Form’s mod slot, the code will automatically unequip Fire Form and turn it into Construct Form’s mod.

    I created the mod menu functions from scratch, including implementing UI assets, implementing drag and drop functionals, and connecting UI with backend codes. n game design, switching between forms and mods has various restrictions; for example, you can’t mod the same form (e.g. construct mod can’t mod construct form), or you can’t equip the same Form twice, etc. I have considered all these edge cases in UI to indicate to players the correct way to equip mods and forms.

    Item Menu

    Item Menu UI will pop out with three item options when the player clears an item room, and they can read through the item description to select one to apply. They simply need to click the card to use the item. If the item pool has less than three available items, it will only spawn two items for selection.
    Both the Mod Menu and Item Menu panels are toggleable.
