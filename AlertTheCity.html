Alert The City (2024)

Play Now
Alert the city is a a 3D top-down traffic simulation and educational game that teaches players about wildfire evacuation strategies. Player act as a community leader to make critical decisions during a wildfire evacuation to avoid loss and ensure safety. Each choice directly impacts the final results of evacuation, offering valuable lessons on emergency preparedness and response. With interactive scenarios and realistic challenges, this game provides a hands-on way to understand wildfire evacuation planning and safety.

What part did I do?

Traffic Simulation & AI System

  • Dynamic Pathfinding: Utilized A* and K-shortest path algorithms for adaptable vehicle navigation, enabling cars to reroute based on traffic congestion and road conditions. A CarAI and CarController manage speed, collision avoidance, and honking responses.
  • Adaptive Road Management: Designed ATC_RoadManager and RoadFixer to configure road types (e.g., intersections, dead-ends) automatically, optimizing traffic flow. The RoadHelper component ensures accurate vehicle placement and smooth path transitions.
  • Car Spawning Variability: Created ATC_CarSpawner to introduce randomness in spawn timing and vehicle types, generating organic traffic patterns and diverse AI behaviors.

Player Movement and Interaction:

  • Finite State Machine for Interactions: Used a Finite State Machine (FSM) for managing the state of interactable objects, ensuring player interactions are only possible under the correct conditions.
  • Input Management: Integrated Unity's Input System for seamless player control, handling both free-walking and grid-based movements within the same game system.
  • Raycasting for Environmental Interactions: Applied 2D Raycasting in the PlayerController for detecting interactable objects, optimizing performance while providing real-time interaction feedback.

Enemy AI and Combat Mechanics:

  • Shared Combat Logic (Inheritance): Utilized inheritance and polymorphism by extending a Damageable class for both player and enemies, allowing for shared functionality while enabling customization of unique behaviors like enemy-specific rewards.
  • Dynamic Enemy Attack Patterns: Managed enemy attack behavior with coroutines, introducing dynamic timing between actions and adapting attack routines based on game progression.
  • Difficulty Scaling: Adjusted enemy stats dynamically based on difficulty (Easy, Mid, Hard, Boss) using scaling logic to maintain challenge across the different areas of the game.

UI and Visual Effects:

  • Prefab-Based UI Design: Built a dynamic UI for cards and inventory using a prefab-based approach, enabling real-time instantiation and updates for card elements (CardUI.cs) during gameplay.
  • Event-Driven UI Updates: Employed event-driven programming to update the UI based on player and enemy status, ensuring real-time feedback for health, combat state, and inventory updates.
  • State-Driven UI Management: Implemented a state-driven UI, which dynamically adjusts based on the battle phases (e.g., disabling the card selection interface during EnemyTurn).

Game and Level Management:

  • Singleton Pattern for Managers: Utilized the Singleton pattern for various management systems (e.g., GameManager, ResourceManager), ensuring global access and consistency throughout the game.
  • Dynamic Scene Loading: Managed scene transitions using Unity's SceneManager, allowing for smooth movement between different game areas while maintaining the player's state and progress.

Resource Management:

  • Weighted Randomization for Card Drops: Applied weighted random algorithms to manage card drop rates, ensuring rare cards have a lower probability of appearing, balancing progression.
  • Data-Driven Card Management: Used ScriptableObjects for a modular, data-driven approach to manage card information, allowing for easy expansion and modification of the card pool.
